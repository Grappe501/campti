# Source chunks (`02_source_chunks`)

**Purpose:** Hold **existing** story material, worldbuilding, and data in **sortable** form without flattening it into a single narrative.

## How this relates to production

1. **Intake (optional but recommended):** drop new material in `intake/` first (`new_unsorted_material.md`, `intake_log.md`) before sorting.
2. Add or move files into the closest topical folder (`story_fragments/`, `worldbuilding/`, etc.).
3. Register each meaningful file in `chunk_index.md` with a one-line description and tags.
4. During chapter work, use `source_matches/` under that chapter to point here — copy short excerpts if needed; prefer paths over duplication.

## Canon status

- Material here may be **draft**, **alternate**, or **superseded**. Do not assume everything is CONFIRMED.
- When something is promoted to canon, update `01_canon/` and optionally add a note in the chunk file header.

## `intake/`

Use `intake/` when you want a **logged inbox** before sorting. See `intake/README.md`.

## `unused_or_unsorted/`

Drop inbox material here first; triage into typed folders and index when you know enough to classify.
